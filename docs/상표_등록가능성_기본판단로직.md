# 상표 등록가능성 기본판단로직 v4.1

## 1. 문서 목적
이 문서는 이 프로젝트의 기본 판단 로직 기준 문서(Source of Truth)다.
코드, UI, PDF 보고서, 테스트는 이 문서의 판단 순서를 따른다.

이번 버전의 핵심은 다음과 같다.
- 지정상품 입력의 source of truth는 `docs/지식재산처_상품분류_니스분류.xlsx`다.
- 지정상품 입력은 `분류 1 -> 분류 2 -> 구체상품군 선택`이 끝나야 확정된다.
- UI는 단순하게 유지하되, 실제 판단은 subgroup / nice class / similarity code 기준으로 시작한다.
- 기존 표장 유사성, 문자열 유사성, 점수 산정 로직은 유지한다.

## 2. 유지하는 기존 축
- 기존 발음/호칭 유사도 판단 로직 유지
- 기존 외관/관념/문자열 유사도 로직 유지
- 기존 KIPRIS 검색 흐름 유지
- 기존 유사군코드 추천/매핑 로직 유지
- 기존 식별력 평가 로직의 기본 축 유지

## 3. 기본 판단 순서
1. 입력 정규화
2. 식별력 판단
3. 사용자가 선택한 니스류/상품군 확정
4. 유사군코드 매핑
5. 상품 유사성 1차 필터
6. 기존 표장 유사성 판단
7. 선행상표 상태 반영
8. 혼동 가능성 종합
9. 최종 점수/설명 생성

## 4. 지정상품 입력 구조

### 4-1. UI 입력 순서
- 분류 1: 제품 또는 서비스
- 분류 2: 요약형 UI 카테고리
- 구체상품군: 실제 subgroup 선택

분류 2는 모바일형 요약 카테고리일 뿐이다.
실제 지정상품 범위는 구체상품군 선택이 끝난 뒤에 확정된다.

### 4-2. 내부 정규화 필드
분석 엔진은 아래 값을 기본 입력으로 받는다.
- `selected_kind`
- `selected_groups`
- `selected_subgroups`
- `selected_nice_classes`
- `selected_similarity_codes`
- `selected_keywords`
- `specific_product_text`

## 5. 권리범위 판단 시작점

### 5-1. 기본 원칙
- 권리범위 판단은 사용자가 선택한 subgroup와 연결된 니스류를 중심으로 시작한다.
- 같은 류라고 무조건 충돌로 보지 않는다.
- 상품군과 유사군코드가 멀면 동일 표장이라도 직접 충돌로 보지 않는다.
- 타 류라도 유사군코드가 같거나 예외적 혼동 우려가 있으면 검토군에 올릴 수 있다.

### 5-2. 유사군코드 우선 원칙
- 지정상품의 동일·유사 여부는 원칙적으로 유사군코드를 기준으로 본다.
- 동일 유사군코드가 가장 강한 기준이다.
- 동일 니스류는 보조 구조화 기준이다.
- 동일 니스류 안에서도 유사군코드가 다르면 보조 검토군으로만 본다.
- 사용자가 실제로 선택한 유사군코드가 상품 유사성 판단의 최우선 입력이다.

## 6. 상품 유사성 필터 구조
선행상표 후보는 아래 네 그룹으로 나눈다.

### 6-1. `exact_scope_candidates`
- 동일 유사군코드
- 선택된 subgroup와 실질적으로 맞닿는 후보
- 최종 점수 직접 반영의 최우선 대상

### 6-2. `same_class_candidates`
- 동일 니스류이나 유사군코드는 다름
- subgroup 문맥은 일부 맞닿을 수 있으나 동일 범위로 단정하지 않음

### 6-3. `related_market_candidates`
- 타 류 또는 상품-서비스업 사이지만 예외적 혼동 우려가 있는 경우

### 6-4. `irrelevant_candidates`
- 타 류, 타 코드, 상품군 관련성 없음
- 검색 결과에는 표시할 수 있으나 최종 점수 직접 반영 금지

## 7. UI 단순화와 판단 로직의 관계
- 분류 2 화면에서는 긴 니스류 설명을 숨긴다.
- 구체상품군은 별도 단계에서만 보여준다.
- 그러나 내부 판단은 여전히 subgroup / nice class / similarity code를 그대로 쓴다.
- 즉 UI 단순화는 표시 방식의 변경이지, 판단 입력의 축소가 아니다.

## 8. 선행상표 상태 반영 원칙
- `live_blockers`: 등록, 출원, 공고, 심사중
- `historical_references`: 거절, 포기, 취하, 소멸, 말소, 무효

`live_blockers`만 최종 점수에 직접 반영한다.
`historical_references`는 참고자료 또는 보조 경고로만 다룬다.

## 9. 최종 점수 캘리브레이션
- 상품 유사성 필터를 통과한 선행상표가 없으면 상대적 거절사유 리스크는 매우 낮게 본다.
- 동일 유사군코드 + 높은 표장 유사도 + 등록/출원 상태면 강한 위험으로 본다.
- 거절/포기/취하/소멸 상표는 직접 감점하지 않는다.
- 타 류/타 상품군/타 코드 후보만으로 점수가 과도하게 낮아지지 않게 한다.
