# 상표 등록가능성 기본판단로직 v3.0

## 1. 문서 목적
이 문서는 이 프로젝트의 기본 판단 로직 기준 문서(Source of Truth)다.
코드, UI, PDF 보고서, 테스트는 이 문서의 판단 순서를 따른다.

이번 버전의 핵심은 다음 다섯 가지다.
- 사용자의 상품 입력 source of truth를 니스분류로 고정한다.
- 지정상품의 동일·유사 여부는 류번호만이 아니라 유사군코드를 우선 기준으로 본다.
- 기존 표장 유사성 판단 로직은 유지하되, 상품 범위 필터를 통과한 후보에 대해 강하게 적용한다.
- 선행상표 상태를 실질 장애물과 역사적 참고자료로 분리한다.
- 거절상표는 거절이유가 현재 상표와 직접 관련될 때만 보조 경고로 반영한다.

## 2. 유지하는 기존 축
- 기존 발음/호칭 유사도 판단 로직 유지
- 기존 외관/관념/문자열 유사도 로직 유지
- 기존 KIPRIS 검색 흐름 유지
- 기존 유사군코드 추천/매핑 로직 유지
- 기존 Streamlit step UI의 큰 흐름 유지
- 기존 식별력 평가 로직의 기본 축 유지

기존 표장 유사성 판단 로직은 유지하되, 최종 충돌 판단은 사용자가 선택한 니스분류, 상품군, 유사군코드 및 선행상표 상태를 종합하여 판단한다.

## 3. 기본 판단 순서
1. 입력 정규화
2. 식별력 판단
3. 지정상품 범위 확정
4. 상품 유사성 필터
5. 표장 유사성 판단
6. 선행상표 상태 반영
7. 혼동 가능성 종합
8. 최종 점수/설명 생성

## 4. 니스분류 기반 입력 구조

### 4-1. Source of Truth
- goods는 제1류~제34류다.
- services는 제35류~제45류다.
- 분류 1 / 분류 2 / 상품군은 UX 레이어다.
- 실제 저장과 판단은 니스류 번호, 상품군 문맥, 유사군코드 기준으로 한다.

### 4-2. 내부 정규화 필드
분석 엔진은 아래 값을 기본 입력으로 받는다.
- `selected_kind`: `goods` 또는 `services`
- `selected_groups`: 선택한 UI 대분류 목록
- `selected_subgroups`: 선택한 상품군 목록
- `selected_nice_classes`: 실제 니스류 번호 목록
- `selected_similarity_codes`: 사용자가 선택한 유사군코드 목록
- `selected_keywords`: 상품군 문맥 키워드
- `specific_product_text`: 사용자가 직접 입력한 구체 상품/서비스명

### 4-3. 저장 원칙
- 사용자가 제품을 선택하면 goods(1~34류)만 선택 가능하다.
- 사용자가 서비스를 선택하면 services(35~45류)만 선택 가능하다.
- 복수 상품군 선택 시 연결된 니스류는 dedupe 한다.
- 복수 상품군 선택 시 연결된 유사군코드는 dedupe 한다.

## 5. 지정상품 범위와 권리범위 판단

### 5-1. 기본 원칙
- 권리범위 판단은 사용자가 선택한 니스류, 상품군, 유사군코드 안에서 우선 검토한다.
- 동일 표장이라도 상품류/상품군/유사군코드가 실질적으로 멀면 직접 충돌로 보지 않는다.
- 동일 또는 가까운 상품군 범위 안에 있는 경우에만 표장 유사성 판단을 강하게 반영한다.

### 5-2. 유사군코드 우선 원칙
- 지정상품의 동일·유사 여부는 류번호만이 아니라 원칙적으로 유사군코드를 기준으로 본다.
- 동일 유사군코드가 가장 강한 기준이다.
- 동일 니스류는 보조 구조화 기준이다.
- 니스류만 같고 유사군코드가 다르면 보조 검토군으로만 본다.

## 6. 상품 유사성 필터 우선 구조
선행상표 후보는 아래 네 그룹으로 나눈다.

### 6-1. `exact_scope_candidates`
- 동일 유사군코드
- 동일 또는 강하게 연결된 상품군
- 최종 점수 직접 반영의 최우선 대상

### 6-2. `same_class_candidates`
- 동일 니스류이나 유사군코드는 다름
- 보조 검토군

### 6-3. `related_market_candidates`
- 타 류 또는 상품-서비스업 사이지만 출처 혼동 가능성이 있는 경우
- 예외 검토군

### 6-4. `irrelevant_candidates`
- 타 류, 타 코드, 상품군 관련성 없음
- 검색 결과에는 표시할 수 있으나 최종 점수 직접 반영 금지

## 7. 상품-서비스업 간 예외 판단

### 7-1. 원칙
- 상품과 서비스업 간은 원칙적으로 유사도가 낮다.
- 따라서 goods/services가 다르면 바로 강한 감점으로 가지 않는다.

### 7-2. 예외 요소
아래를 종합 고려해 보조 검토군으로 올릴 수 있다.
- 동일 사업자가 통상 함께 제공하는지
- 상품의 용도와 서비스 내용이 맞닿는지
- 판매장소/제공장소가 맞닿는지
- 수요자 범위가 맞닿는지
- 양 표장이 전체적으로 동일 또는 극히 유사한지

### 7-3. 소프트웨어(G390802) 예외
소프트웨어(G390802 포함)와 아래 서비스는 별도 예외 검토 대상이다.
- 제42류 소프트웨어 개발, SaaS, 플랫폼 서비스
- 제35류 일부 플랫폼 판매중개/운영 서비스
- 제38류 일부 통신/플랫폼 서비스

다만 이 경우에도 양 표장이 전체적으로 동일 또는 극히 유사한 경우에 한해 강한 예외 검토를 적용한다.

## 8. 표장 유사성 판단
- 기존 발음/호칭, 외관, 관념, 문자열 유사도 로직은 유지한다.
- 다만 상품 유사성 필터를 통과한 후보에 대해서만 최종 감점에 강하게 반영한다.
- 필터를 통과하지 못한 후보는 참고용 계산만 가능하고 최종 점수 직접 반영은 금지한다.

### 8-1. 완전 동일 표장 처리
- 대소문자 차이만 있는 동일 표장은 `mark_identity = exact`로 분류한다.
- `exact`이고 상품 유사성 필터를 통과하면 후보 카드의 표장 유사도는 100%다.
- `exact + 동일 유사군코드 + live_blocker`면 후보 카드 혼동위험은 최소 95 이상으로 반영한다.
- `exact + historical_reference`면 카드에는 경고를 띄우되 최종 점수 직접 감점은 금지한다.

## 9. 선행상표 상태별 반영 원칙

### 9-1. `live_blockers`
- 등록
- 출원
- 공고
- 심사중

이 그룹만 최종 점수에 직접 반영한다.

### 9-2. `historical_references`
- 거절
- 포기
- 취하
- 소멸
- 말소
- 무효

이 그룹은 원칙적으로 최종 점수에 직접 반영하지 않는다.
후보 카드 또는 설명 텍스트에서 참고자료, 역사적 참고자료, 보조 경고로만 다룬다.

## 10. 거절 상표 처리 원칙

### 10-1. 기본 원칙
- 거절상표는 최종 점수 직접 감점 금지
- `top_prior` 우선순위는 `live_blocker`보다 낮춤
- 동일 표장이라도 거절/취하/포기/소멸이면 직접 장애물로 보지 않고 참고자료로 처리

### 10-2. 거절결정문 분석 구조
가능한 경우 아래 정보를 저장한다.
- `reason_summary`
- `cited_marks`
- `refusal_core`
- `refusal_basis`
- `weak_elements`
- `current_mark_relevance`

### 10-3. 보조 경고로 반영하는 기준
- 거절결정문 또는 통합행정정보에서 거절이유가 확보된 경우에만 본다.
- 현재 상표와 직접 관련된 핵심 요부/호칭 충돌이 반복되는 경우에만 보조 경고로 반영한다.
- 이 경우에도 `live_blocker`와 같은 강한 감점은 금지한다.

## 11. 최종 점수 캘리브레이션
최종 점수는 아래 3축을 합성한다.
- 식별력 리스크
- 상품 유사성/권리범위 리스크
- `live_blockers` 기준 선행상표 충돌 리스크

반드시 아래 원칙을 지킨다.
- 상품 유사성 필터를 통과한 선행상표가 없으면 상대적 거절사유 리스크는 매우 낮게 본다.
- 동일 유사군코드 + 높은 표장 유사도 + 등록/출원 상태면 강한 위험으로 본다.
- 거절/포기/취하/소멸 상표는 직접 감점하지 않는다.
- 타 류/타 상품군/타 코드 후보만으로 점수가 과도하게 낮아지지 않게 한다.

## 12. UI/결과 표시 원칙
반드시 아래 항목을 구분해서 보여준다.
- 선택한 분류 1: 제품/서비스
- 선택한 분류 2
- 선택한 상품군
- 연결 니스류
- 연결 유사군코드
- 전체 검색 건수
- 상품 유사성 필터 통과 건수
- 실질 충돌 후보 건수
- 제외 후보 건수

또한 아래 문구 구조를 유지한다.
- 완전 동일한 선행상표가 있으나 현재 상태가 거절/취하/포기/소멸인 경우, 원칙적으로 직접 장애물로 보지 않고 참고자료로만 봅니다.
- 등록 또는 출원 상태의 동일/유사 상표는 실질 장애물로 평가합니다.
- 거절 상표는 거절이유의 핵심이 현재 상표와 직접 관련되는 경우에만 보조 경고로 반영합니다.

## 13. 예시

### 13-1. 제품 20류 가구
- 선택값: goods / 가구 / 제20류 / `G2001`
- 동일 유사군코드의 등록·출원 상표가 있으면 최고 위험군이다.
- 같은 제20류라도 `G2002` 등 다른 코드면 중간 위험군 또는 보조 검토군이다.

### 13-2. 9류 소프트웨어
- 선택값: goods / 소프트웨어·앱 / 제9류 / `G390802`, `G0901`
- 동일 코드의 live blocker가 있으면 최종 점수를 강하게 낮춘다.
- 제42류 SaaS/플랫폼 서비스와는 상품-서비스업 예외군으로 별도 검토한다.

### 13-3. 서비스 41/42류
- 선택값: services / 교육 또는 IT 서비스 / 제41류·제42류
- 동일 유사군코드 또는 동일 니스류의 live blocker를 우선 검토한다.
- 동일 표장이라도 제류와 코드가 멀면 직접 충돌로 보지 않는다.

### 13-4. 거절상표 보조 참고 사례
- FlexAiCam 거절은 `Flex`가 약한 요소이고 `AiCam`이 핵심 요부인 사례로 구조화한다.
- 인용상표는 `에이캠`, `에이켐`, `ICAM`으로 저장한다.
- `LexAI` 평가에서는 직접 관련성이 낮으므로 최종 점수 직접 감점 없이 역사적 참고자료로만 처리한다.
