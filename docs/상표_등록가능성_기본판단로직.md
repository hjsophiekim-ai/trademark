# 상표 등록가능성 기본 판단 로직

## 1. 검토 진입 조건
검토 진입의 기준은 `selected_subgroup_ids`다.

- `selected_subgroup_ids` 1개 이상 존재
- `derived_nice_classes` 계산 완료
- `derived_similarity_codes` 계산 완료

아래 조건은 검토 진입 조건에서 제거한다.

- `selected_codes` 존재 여부
- 사용자 유사군코드 직접 선택 여부

## 2. 사용자 흐름
사용자-facing 흐름은 아래로 고정한다.

1. 상표명 입력
2. 상품범위 선택
3. 검토 실행
4. 결과
5. 개선방안

상품범위 선택 내부 흐름은 아래와 같다.

1. 분류 1 선택
2. 분류 2 선택
3. 구체상품군 선택

## 3. 시스템 내부 파이프라인
구체상품군 선택 이후 검토 로직은 아래 순서로 연결된다.

1. `selected_subgroup_ids`
2. `selected_subgroup_labels`
3. `derived_nice_classes`
4. `derived_similarity_codes`
5. KIPRIS 검색
6. 상품 유사성 필터
7. 기존 표장 유사성 판단
8. 선행상표 상태 반영
9. 결과 출력
10. 개선방안 생성

## 4. 유사군코드 도출 원칙
유사군코드의 source of truth는 `docs/상품유사군코드.xlsx`다. 구체상품군과 니스류의 source of truth는 `docs/지식재산처_상품분류_니스분류.xlsx`다.

- 사용자는 유사군코드를 직접 선택하지 않는다.
- 시스템이 선택된 구체상품군과 추가 입력된 구체 상품명/서비스명을 보고 예규상 가장 적합한 실제 유사군코드를 자동 도출한다.
- `S3601`, `S3602`, `S3603` 같은 가상코드는 사용하지 않는다.
- 동일한 subgroup에 무관한 코드 여러 개를 과도하게 붙이지 않고, 우선 1개의 대표 코드를 선택한다.

매칭 우선순위는 아래 순서를 따른다.

1. `exact_label_match`
2. `normalized_semantic_match`
3. `keyword_dictionary_match`
4. `same_class_fallback`

도출 결과에는 내부적으로 아래 감사 정보가 함께 남아야 한다.

- `candidate_similarity_codes`
- `chosen_similarity_codes`
- `match_reason`
- `match_confidence`
- `fallback_used`

## 5. 핵심 원칙
- `selected_subgroup_ids`가 실제 사용자 선택의 source of truth다.
- `derived_similarity_codes`는 시스템이 자동 계산한 internal derived value다.
- 레거시 `selected_codes`는 내부 호환용으로만 남길 수 있으나, Step 전환 조건과 검토 진입 조건에는 사용하지 않는다.
- 동일 니스류라고 해서 무조건 충돌로 보지 않고, 도출된 유사군코드와 상품 문맥을 함께 반영한다.

제36류는 아래 기준을 우선 적용한다.

- `금융, 통화 및 은행업` -> `S0201`
- `보험서비스업` -> `S0301`
- `재무평가/재무관리/재무상담` 계열 -> `S120401`
- `부동산업` -> `S1212`
- 그 외 명시 매칭 실패 -> `S173699`

## 6. 결과 화면에 보여줄 선택 정보
결과와 보고서에는 아래 선택 정보를 일관되게 표시한다.

- 선택한 분류 1
- 선택한 분류 2
- 선택한 구체상품군
- 도출된 니스류
- 내부 도출 유사군코드

## 7. 검토 실행 가능 조건
`검토 실행` 버튼 활성화 조건:

- `len(selected_subgroup_ids) >= 1`

즉, 구체상품군만 확정되면 검토는 실행 가능해야 하며, 사용자가 유사군코드를 다시 선택할 필요는 없다.
