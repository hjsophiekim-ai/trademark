반드시 수정해야 할 핵심 문제:
- 현재는 상품 유사성 필터보다 이름 유사도 감점이 먼저 강하게 들어가서,
  사용자가 3류를 선택했는데 25류 선행상표만 있어도 등록 가능성이 과도하게 낮아질 수 있다.
- 이 문제를 고쳐라.
- 앞으로는 반드시 아래 순서로 판단하게 만들어라.

기본 판단 순서:
1. 입력 정규화
2. 식별력(절대적 거절사유) 판단
3. 선행상표 후보의 상품 유사성 필터링
   - 1차: 동일 유사군코드
   - 2차: 동일 류
   - 3차: 타 류 예외군(경제적 견련성, 저명성, 기만 우려 등)
4. 그 후에만 표장 유사성 판단
   - 외관
   - 호칭/발음/명칭
   - 관념
5. 상품 유사성과 표장 유사성을 함께 종합하여 혼동 가능성 판단
6. 최종 점수와 설명 생성

반드시 반영할 세부 원칙:
A. 상품 유사성 필터가 먼저다.
- 선행상표 전체를 바로 감점에 넣지 마라.
- 먼저 사용자가 선택한 류, 구체 상품명, 유사군코드를 기준으로 후보를 걸러라.

B. 같은 류라고 무조건 유사상품은 아니다.
- 동일 류라도 상품 성질, 용도, 판매채널, 수요자 범위가 다르면 감점을 약하게 하거나 보조 검토군으로 둬라.

C. 다른 류라고 무조건 무관하지도 않다.
- 유사군코드가 같거나 경제적 견련성이 강하면 예외적으로 검토하라.
- 다만 예외 사정이 없으면 타 류, 타 코드 후보는 강한 감점에 반영하지 마라.

D. 0% 또는 등록 불가 판단은 엄격히 제한하라.
- 전혀 다른 상품영역의 상표가 있다는 이유만으로 0%를 주지 마라.
- 0%에 가까운 평가는 아래 경우에 한정하라:
  1) 식별력 자체가 매우 약하고
  2) 동일 또는 매우 유사한 유사군코드 내에서
  3) 표장 유사도도 매우 높고
  4) 혼동 가능성이 큰 경우

E. 절대적 거절사유와 상대적 거절사유를 분리하라.
- 식별력(제33조) 판단과 선행상표 충돌(제34조) 판단을 별도 구조로 반환하라.
- 기존 score 필드는 유지하되, 내부 판단은 분리해서 설명 가능하게 하라.

구현 요구사항:
1. docs/상표_등록가능성_기본판단로직.md 내용을 기준으로 로직을 재정렬해라.
2. 기존 유사도 계산 함수는 되도록 유지하되, 적용 순서를 바꿔라.
3. scoring.py 에서는 상품 유사성 필터를 통과하지 않은 후보 때문에 과도한 감점이 일어나지 않게 수정해라.
4. 현재 selected_fields, selected_codes, specific_product 정보를 상품 유사성 판단에 적극 반영해라.
5. KIPRIS 결과를 사용할 때, 동일 코드/동일 류/예외군 구조로 나누어 처리해라.
6. 결과 화면(step 4)에는 아래 내용을 분리해 보여줘:
   - 식별력 판단
   - 상품 유사성 검토 결과
   - 표장 유사성 검토 결과
   - 혼동 가능성 종합
